## 2. 朝向系统（全新设计）

### 2.1 核心概念

每个单位拥有一个**独立的朝向状态**（360° 方向值）。朝向改变有固定角速度（掉头速度）。所有行为（移动、攻击）都受朝向约束：单位需要先朝向目标方向，才能有效执行行为。

### 2.2 数据结构

```rust
/// 单位朝向（所有士兵实体必须挂载）
#[derive(Component)]
pub struct FacingDirection {
    pub angle: Fixed,  // 当前朝向角度（0-360°，定点数）
}
```

### 2.3 基础数值

| 参数 | 值 | 说明 |
|------|-----|------|
| 转向速度 | 360°/秒 (20 ticks/圈) | 所有兵种相同 |
| 移动偏差减速 | 线性：偏差 90° = 50% 速度 | 偏差越大越慢 |
| 攻击转向 | 自动转向目标，不需完全对准 | 可边转边攻击 |

### 2.4 朝向与移动

**移动流程：**
1. 计算目标方向（当前位置 → 目标位置的角度）
2. 以固定角速度（360°/s）逐步转向目标方向
3. 计算朝向偏差（当前朝向与目标方向的夹角）
4. 移动速度 = 基础速度 × (1 - 偏差/180°)
   - 偏差 0° → 100% 速度
   - 偏差 90° → 50% 速度
   - 偏差 180° → 0% 速度（原地掉头）

**效果：**
- 单位直行时速度正常
- 转弯时自动减速
- 急停掉头（180°）需要先花时间转身

### 2.5 朝向与攻击

**攻击流程：**
1. 确定攻击目标方向
2. 自动开始转向目标方向（同时可以发起攻击，不需完全对准）
3. 攻击冷却照常计时

**非骑兵兵种边走边打时：**
- 攻击瞬间有一个 **0.15 秒 (3 ticks) 的前摇停顿**
- 停顿期间单位不移动
- 前摇结束后恢复移动
- 类似 MOBA 游戏（英雄联盟/王者荣耀）的攻击动画机制

**骑兵边走边打时：**
- 完全无停顿，边跑边砍
- 有极短的攻击动画（0.1s），但纯视觉效果，不影响移动速度

### 2.6 朝向与受击

**受击不影响朝向**。朝向只由移动和攻击目标决定。

### 2.7 朝向与盾牌

盾牌正面 120° 的判定基于 `FacingDirection.angle`。详见盾牌系统章节。

## 3. 盾牌系统（全新设计）

### 3.1 核心概念

盾牌是一个**独立物品**，拥有自己的 HP（白色血条，区别于士兵的绿色血条）。盾牌可以被捡起、掉落、破碎，并在城池光环范围内回复 HP。

### 3.2 数据结构

**simulation 层新增/修改组件：**

```rust
/// 盾牌物品数据（挂载在士兵实体上，或作为 DroppedShield 的子数据）
#[derive(Component)]
pub struct ShieldItem {
    pub hp: u32,
    pub max_hp: u32,
}

/// 盾牌状态（仅持有盾牌的士兵有此组件）
#[derive(Component)]
pub enum ShieldState {
    Normal,     // 普通状态
    Blocking,   // 手动格挡中（仅步兵可切换）
}

/// 地面上的掉落盾牌实体
#[derive(Component)]
pub struct DroppedShield {
    pub shield: ShieldItem,
    pub position: FixedVec2,
    pub drop_tick: u32,                   // 掉落时的 tick（用于 30s 计时）
    pub owner_faction: Option<Faction>,   // 掉落者阵营（渲染用）
}
```

**士兵组件变更：**
- `ShieldComponent` → 改为 `Option<ShieldItem>`（士兵可能没有盾牌）
- 保留 `ShieldState` 组件（仅持有盾牌的士兵有意义）

### 3.3 基础数值

| 参数 | 值 | 来源 |
|------|-----|------|
| 盾牌初始 HP | 1500 | 需新增配置项 |
| 掉落存活时间 | 30 秒 (600 ticks) | 需新增配置项 |
| 消失动画时长 | 3 秒 (60 ticks) | 需新增配置项 |
| 被动格挡概率 | 40% | 需新增配置项 |
| 手动格挡移速 | 80 → 15 | 需新增配置项 |
| 手动格挡攻速 | 0.5s → 3s (10 → 60 ticks) | 需新增配置项 |
| 正面判定角度 | 120° | 需新增配置项 |
| 拾取半径 | 等于碰撞半径 | - |

### 3.4 兵种持有规则

| 兵种 | 能否持有 | 手动格挡 | 被动格挡 | 出生状态 |
|------|---------|---------|---------|---------|
| 步兵 | ✅ | ✅ 移速→15, 攻速→3s, 正面 120° 全额挡伤 | ✅ 40% | 自带满 HP 盾牌 |
| 民兵 | ✅ | ❌ | ✅ 40% | 无盾 |
| 骑兵 | ✅ | ❌ | ✅ 40% | 无盾 |
| 弓兵 | ❌ | ❌ | ❌ | 无盾 |

### 3.5 系统流程

#### 3.5.1 出生

步兵从城池出生时，自动获得 `ShieldItem { hp: 1500, max_hp: 1500 }` 和 `ShieldState::Normal`。

#### 3.5.2 被动格挡（所有持盾士兵）

每次受到伤害时：
1. 独立判定 40% 概率
2. 命中 → 伤害转移到盾牌 HP（盾牌 HP 扣减，士兵 HP 不扣）
3. 未命中 → 正常扣士兵 HP

**被动格挡统一替代原有 `intercept_chance` 机制**，只要格挡触发，必拦截。

#### 3.5.3 手动格挡（仅步兵）

玩家发送 `SetShield` 命令切换 `ShieldState::Normal` ↔ `ShieldState::Blocking`。

**Blocking 状态下：**
- 移速: 80 → 15
- 攻速: 10 ticks → 60 ticks
- 正面 120° 扇形内的所有伤害: 100% 由盾牌承担
- 非正面伤害: 正常扣士兵 HP（不经过被动格挡判定）

**角度判定：**
- 以 `FacingDirection.angle` 为中心，左右各 60° 的扇形（共 120°）
- 攻击来源在扇形内 → 全额由盾牌承担
- 攻击来源在扇形外 → 正常伤害

#### 3.5.4 盾牌破碎

当盾牌 HP ≤ 0 时：
- 移除士兵的 `ShieldItem` 和 `ShieldState` 组件
- 盾牌消失，不掉落

#### 3.5.5 死亡掉落

当持有盾牌的士兵死亡时：
1. 在士兵位置生成 `DroppedShield` 实体
2. `DroppedShield` 包含原盾牌的 HP 数据
3. 记录 `drop_tick` 用于计时

#### 3.5.6 掉落盾牌生命周期

```
tick 0        : 盾牌掉落，开始存在于地面
tick 600 (30s): 开始 3 秒消失动画
tick 660 (33s): 动画结束，盾牌实体销毁
```

#### 3.5.7 拾取检测

每 tick，检测所有士兵附近的 `DroppedShield`。检测距离 = 碰撞半径。

```
if 士兵是弓兵:
    跳过
if 士兵无盾:
    直接拾取（移除 DroppedShield，给士兵添加 ShieldItem）
if 士兵有盾:
    if 地上盾牌 HP > 士兵当前盾牌 HP:
        拾取新盾牌，丢弃旧盾牌（旧盾牌不掉落，直接消失）
    else:
        跳过
```

#### 3.5.8 城池光环回复

盾牌 HP 与士兵 HP 一样，在城池光环范围内受到相同的回复效果。
- 回复量: `base_heal + city_level * per_level_heal`（与士兵 HP 回复相同）
- 回复上限: 不超过 `max_hp`

### 3.6 content/combat.ron 新增配置

```ron
shield: (
    speed_penalty: 15,                // 手动格挡时移速（80→15）
    attack_speed_penalty: 60,         // 手动格挡时攻击间隔（ticks）
    passive_block_chance: 0.40,       // 被动格挡概率
    frontal_angle: 120,               // 正面判定角度（度）
    initial_hp: 1500,                 // 盾牌初始 HP
    drop_survive_ticks: 600,          // 掉落存活时间（ticks）
    disappear_animation_ticks: 60,    // 消失动画时长（ticks）
),
```

**移除项：** `intercept_chance`、`damage_reduction`（被新系统替代）

## 4. 弓兵行为修正

### 4.1 Multi-shot（新增）

**触发条件：**
- 弓兵攻击冷却结束时，按概率判定是否触发 multi-shot
- 概率公式: `base_chance + level * per_level_bonus`，上限 `max_chance`
- 当前配置: `base_chance: 0.05, per_level_bonus: 0.005, max_chance: 0.30`

**触发行为：**
- 从射程内随机选择 2-5 个不同敌人（每个敌人最多被选中一次）
- 对每个目标各射一箭（独立的 Arrow 实体）
- 所有箭矢共享同一个发射位置（弓兵位置）

**冷却：**
- 触发 multi-shot 后，正常进入 0.5s 攻击冷却

**目标选择：**
- 从射程内的所有敌人士兵中随机选择
- 如果射程内敌人数不足 multi-shot 数量，则射出对应数量的箭
- 如果射程内没有敌人，不触发 multi-shot

### 4.2 追击行为（修改）

**当前行为：**
- 弓兵在 `Fighting` 状态下完全不移动

**修改后行为：**
- 弓兵在 `Fighting` 状态下，如果当前目标离开射程：
  1. 弓兵向目标方向移动，追击到射程边缘
  2. 移动过程中如果目标重新进入射程，继续射击
  3. 边走边射击（参见边走边打机制）

**实现方式：**
- 修改 `soldier_movement_system` 中的弓兵特殊逻辑
- 移除"弓兵在 Fighting 状态下跳过移动"的硬编码
- 改为：弓兵在 Fighting 状态下，检查目标是否在射程内
  - 在射程内 → 不移动，继续射击
  - 不在射程内 → 向目标移动，边走边射（有 0.15s 前摇停顿）

### 4.3 移除近距离伤害惩罚

**移除内容：**
- `combat.ron` 中的 `archer_melee_range: 50` 和 `archer_melee_damage_mult: 0.85`
- 代码中相关的近距离判定逻辑

## 5. 边走边打机制

### 5.1 规则总览

| 兵种 | 攻击时移动 | 前摇停顿 | 说明 |
|------|-----------|---------|------|
| 骑兵 | ✅ 不减速 | 无（0.1s 动画，纯视觉） | 边跑边砍 |
| 步兵 | ✅ 减速 | 0.15s (3 ticks) | 攻击瞬间短暂停下 |
| 民兵 | ✅ 减速 | 0.15s (3 ticks) | 攻击瞬间短暂停下 |
| 弓兵 | ✅ 减速 | 0.15s (3 ticks) | 攻击瞬间短暂停下 |

### 5.2 实现逻辑

**非骑兵兵种：**
1. 攻击冷却归零时，进入前摇状态
2. 前摇期间（3 ticks）：单位不移动（但朝向仍在调整）
3. 前摇结束：发射攻击（近战伤害判定 / 弓兵射箭），进入冷却
4. 前摇结束后：恢复移动

**骑兵：**
1. 攻击冷却归零时，直接发射攻击
2. 无停顿，移动速度不变
3. 播放 0.1s 攻击动画（纯视觉）

### 5.3 与朝向系统的交互

- 攻击前摇期间，单位仍然可以转向（朝向调整不受前摇影响）
- 非骑兵兵种：前摇结束后，单位朝向已经调整了一部分，移动速度基于当前朝向偏差计算

## 8. 系统交互关系图

```
朝向系统 (FacingDirection)
    │
    ├── 移动系统 (Movement)
    │   └── 偏差减速: speed × (1 - angle_diff/180°)
    │
    ├── 攻击系统 (Attack)
    │   ├── 自动转向目标（不需完全对准）
    │   └── 非骑兵: 0.15s 前摇停顿
    │
    └── 盾牌系统 (Shield)
        └── 正面 120° 判定基于 FacingDirection.angle

骑兵闪避 (Dodge)
    │
    └── 盾牌被动格挡 (Passive Block)
        └── 闪避优先判定，闪避未命中后判定被动格挡
```
