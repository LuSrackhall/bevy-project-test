# 城池争霸 — RTS 游戏设计规格

## 概述

基于 Bevy 引擎的极简 RTS 游戏。圆圈代表城池，城池自动产出士兵，玩家通过框选指挥士兵攻占敌方城池。支持单人对战 AI，优先完成后可扩展为多人联机。

## 技术选型

- **引擎**：Bevy 0.18.1
- **2D 绘制**：bevy_prototype_lyon 0.14（城池圆环、血条、框选）
- **随机数**：rand 0.8
- **架构**：纯 ECS，每个模块独立 Plugin，通过 Event + Resource 通信

## 模块划分

```
src/
├── main.rs              # App 入口，注册所有插件
├── core/                # 核心常量、兵种定义
├── camera/              # 摄像机插件（拖拽平移 + 缩放）
├── map/                 # 地图生成插件
├── city/                # 城池插件（Component、血量、等级、归属、产兵）
├── soldier/             # 士兵插件（Component、移动、战斗状态）
├── ai/                  # 人机 AI 插件
├── ui/                  # UI 插件（面板、框选、菜单）
├── combat/              # 战斗系统（弓箭、伤害、闪避、减速）
└── game/                # 游戏流程（状态机、胜利判定）
```

## 一、兵种系统

### 士兵经验与升级

所有兵种均可通过击杀敌方士兵获得经验，经验值满 100 即升级：
- 每升 1 级：最大生命值 +30，攻击力 +6
- 等级无上限
- 4 级以上（含 4 级）的士兵获得 **10% 吸血效果**（造成伤害的 10% 转化为自身生命恢复）

### 属性表（基础值，等级 1）

| 属性 | 民兵 | 步兵 | 弓兵 | 骑兵 |
|------|------|------|------|------|
| 生命值 | 100 | 100 | 100 | 140 |
| 攻击力 | 16 | 20 | 20 | 20 |
| 移动速度 | 80 px/s | 80 px/s | 80 px/s | 200 px/s |
| 攻击范围 | 近战(30px) | 近战(30px) | 远程(380~600px) | 近战(30px) |
| 攻击间隔 | 0.5s | 0.5s | 0.5s | 0.5s |
| 产出速度倍率 | 1.0x | 0.5x | 0.5x | 0.3x |

### 特殊效果

- **民兵**：无特殊效果
- **步兵**：
  - **盾牌系统**：盾牌为独立可消耗物品，步兵默认携带
    - 盾牌 HP = 1500，独立于步兵生命值，受到伤害时优先扣除盾牌 HP
    - 盾牌 HP 归零后盾牌被击毁，步兵失去盾牌加成
    - **被动格挡**：盾牌存在时，所有伤害自动减少 40%（无论正背面）
    - **手动格挡**：框选步兵后可切换手动格挡/正常状态
      - 手动格挡时移速变为 15 px/s（固定值，非百分比减速）
      - 手动格挡时获得正面 120 度扇形全额吸收：来自正面 120 度范围内的伤害完全免疫
      - 手动格挡时背面仍享受被动格挡 40% 减伤
      - 批量框选切换规则：若选中步兵中有格挡和正常两种状态，统一全部变为格挡（即用户想全部正常最多需点两次切换，想全部格挡最多一次）
    - **死亡掉落**：步兵死亡时盾牌掉落在地面，成为可拾取物品
    - **拾取**：任何步兵经过掉落的盾牌时自动拾取，恢复盾牌 HP 至满值
- **弓兵**：
  - 远程攻击，射程 380px (Lv.1) → 600px (Lv.4)，线性缩放；生成 Arrow Entity 固定方向飞行，速度 400 单位/秒
  - 命中附带减速效果：首次命中移速 ×0.85（减速 15%），持续 1s；持续时间内再次命中叠加 ×0.9（额外减速 10%），刷新 1s 持续时间
  - 减速叠加公式：`移速 × 0.85 × 0.9^(n-1)`，上限为移速降至原始速度的 35%
  - **多重射击**：有机率同时攻击 2~5 个目标，目标数随机；触发机率随等级增长，最低 5%，最高 30%
- **骑兵**：
  - 闪避机制：`闪避率 = 50% - (1 - 当前血量/最大血量) × 35%`
  - 满血 50%、半血 32.5%、10%血 18.5%
  - 血量 < 10% 时闪避完全失效（0%）
  - 闪避优先级高于被动格挡：若骑兵同时触发闪避与被动格挡，优先生效闪避
  - **无畏效果**：每次闪避成功时触发，持续 2s，期间攻击力 +10，额外获得 15% 吸血效果（与等级吸血叠加）

### 面朝方向系统

所有士兵具有面朝方向属性，影响正面/背面判定和攻速：
- 士兵移动时面朝移动方向
- 士兵攻击时面朝攻击目标
- 面朝方向决定正面免伤区域（步兵手动格挡时生效）
- 背面受到攻击时无免伤效果

### 朝向攻速因子

攻击间隔受朝向偏差角影响，公式为：
```
攻速因子 = 1 + 0.3 × cos(朝向偏差角)
实际攻击间隔 = 基础攻击间隔 × 攻速因子
```
- 正面对敌（偏差角 = 0）：攻速因子 = 1.3，攻击间隔缩短 30%（更快）
- 侧面对敌（偏差角 = 90）：攻速因子 = 1.0，攻击间隔不变
- 背对敌（偏差角 = 180）：攻速因子 = 0.7，攻击间隔延长 30%（更慢）
- 使用 Bhaskara cosine 近似算法计算 cos 值，避免浮点三角函数
- **骑兵例外**：骑兵无前摇，但攻击冷却仍受朝向攻速因子影响

### 攻击前摇机制

近战单位（非骑兵）发起攻击时有前摇停顿：
- 前摇时长：0.15 秒（3 个逻辑 Tick）
- 前摇期间士兵停止移动，前摇完成后才能发起攻击
- 每次攻击都需要前摇，不仅限于从移动切换时
- **骑兵无前摇**：骑兵可以边移动边攻击，不需停顿

### 近战自动扫描

近战系统每 Tick 直接扫描攻击范围内的最近敌人，不依赖 Movement.target：
- 士兵站立不动时也能自动攻击进入范围的敌人
- 士兵移动中也能自动攻击（取决于移动模式，见 MoveTo vs ForceMove）
- 扫描使用平方距离比较，无需开方计算

### MoveTo vs ForceMove

移动命令分为两种模式：
- **MoveTo**（force_move = false）：允许移动中自动攻击。士兵移动时会扫描范围内敌人，发现目标后自动停下攻击
- **ForceMove**（force_move = true）：抑制自动攻击。士兵只执行移动，忽略范围内敌人
- **骑兵例外**：即使 ForceMove 模式下，骑兵仍会自动攻击（边走边打）

### 伤害计算

```
基础伤害 = 攻击者（攻击力 + 等级加成 + 无畏加成）
  [骑兵目标] → roll 闪避 → 成功则伤害 = 0（触发无畏）（闪避优先于格挡）
  [手动格挡步兵正面 120°] → 伤害 = 0（全额吸收）
  [有盾步兵（被动格挡）] → 伤害 × 0.6（40% 减伤），差额从盾牌 HP 扣除
  [无盾步兵] → 伤害 × 1.0（无减伤）

吸血恢复 = 实际造成伤害 × 吸血率
  - 4 级以上士兵基础吸血率: 10%
  - 无畏状态骑兵额外吸血率: +15%（总计 25%，与等级吸血叠加）
```

### 兵种切换

- 所有城池默认可生产全部 4 种兵种，无需解锁
- 用户点击城池 → 底部面板显示 4 个兵种按钮 → 点击切换
- 切换后仅影响后续产出，已存在的士兵保持原兵种
- 每个士兵占用城池人口上限的 1 个位置，不论兵种

## 二、城池系统

### 属性

- CityLevel: 当前等级
- CityMaxLevel: 等级上限（基础 rand(3..7)，浮动 clamp(基础+rand(-1..+2), 1, 10)）
- CityHealth / CityMaxHealth: 血量（MaxHealth = Level × 100）
- CityPopulation / CityMaxPopulation: 人口（MaxPopulation = Level × 12 + rand(Level×2 .. Level×5)）
- CityFaction: Player | Enemy | Neutral
- CitySpawnType: 当前出兵兵种
- CitySpawnTimer: 产兵计时器
- CityPosition: 地图坐标

### 士兵进城（全部即时消耗士兵）

| 场景 | 效果 |
|------|------|
| 进入敌方/中立城池 | 士兵消失，城池扣血 = 士兵攻击力 × 0.5 |
| 进入己方残血城池 | 士兵消失，回复士兵当前生命值的 30% |
| 进入己方满血 + 等级未满 | 士兵消失，增加 MaxHealth × 30% 升级经验。升级经验阈值为 MaxHealth × 100，达到即升级 |

### 城池光环（领域恢复）

- 己方城池周围 180px 范围内，己方士兵每秒恢复生命值
- 恢复量 = 9 + 城池等级（即 Level 1 = 10 HP/s, Level 2 = 11 HP/s, ...）
- 与士兵自身的吸血效果独立计算，可同时生效
- **城池等级与半径**：城池的视觉圆圈半径随等级增长（等级越高圆圈越大），180px 的光环范围从城池边缘起算，因此实际辐射面积随城池升级自然增大
- **出兵口方向加成**：城池出兵口方向（士兵产出后的移动方向）的领域恢复范围延长至 300px，形成"出兵走廊"效果
- 光环之间可重叠，同一士兵同时处于多个己方城池光环内时效果叠加

### 城池攻克

- CityHealth ≤ 0 → Faction 翻转为攻击方
- Level = max(1, Level - 1)（1级城保持1级）
- Health = 新等级 MaxHealth × 20%
- 被攻陷方由该城产出的士兵不死亡，且不占用其他城池名额上限。这些士兵自动变为"流亡士兵"，继续存在于地图上，归属不变，可继续战斗，但没有产出城池（无法补充）

### 阵营分配（对称）

- 中立城池数量 = rand(N × 0.3 .. N × 0.5)
- 玩家城池 = ceil((N - 中立) / 2)
- AI 城池 = ceil((N - 中立) / 2)
- 确保玩家和 AI 至少各 1 座

## 三、地图生成

### 算法

1. 城池总数 N = rand(6..20)
2. 对每座城池：随机坐标在 [margin, map_size - margin] 内
3. 与已放置城池距离 ≥ city_min_distance（不满足则重试最多 100 次）
4. 分配 Level、MaxLevel、Faction
5. 地图默认尺寸 2000×2000 px

## 四、战斗系统

### 战斗流程

```
近战自动扫描：每 Tick 扫描攻击范围内最近敌人（不依赖 Movement.target）
  → 站立不动时也能自动攻击
  → 移动中能否自动攻击取决于移动模式（MoveTo = 允许，ForceMove = 抑制，骑兵例外）

SoldierState = Fighting
  → 面朝方向更新为朝向攻击目标
  → 计算朝向攻速因子 = 1 + 0.3 × cos(朝向偏差角)（Bhaskara 近似）
  → 非骑兵：前摇 0.15s（3 ticks），前摇期间停止移动
  → 骑兵：无前摇，可边移动边攻击
  → 攻击间隔判定（实际间隔 = 基础间隔 × 朝向攻速因子）：
      [骑兵目标] → roll 闪避 → 成功 → 无伤害 + 触发无畏（持续 2s）（闪避优先于格挡）
      [手动格挡步兵正面 120°] → 伤害 = 0（全额吸收）
      [有盾步兵（被动格挡）] → 伤害 × 0.6，差额从盾牌 HP 扣除
      [弓兵] → 随机触发多重射击（2~5 目标，机率 = clamp(5% + 等级/2, 5%, 30%)）
             → 生成 1~N 个 Arrow Entity（方向有散布：65% 完美瞄准，35% 随机 0.1-10° 偏移，飞行速度 400 单位/秒）
             → Arrow 命中目标时：
                 [手动格挡步兵正面 120°] → 伤害 = 0
                 [有盾步兵（被动格挡）] → 伤害 × 0.6，差额从盾牌 HP 扣除
                 [其余] → 结算伤害 + 减速 debuff
      [其余目标] → 计算伤害 → 扣除血量
  → 目标血量 ≤ 0 → 击杀者获得经验（满 100 升级）
                 → 盾牌掉落（若目标为有盾步兵）
                 → 目标 Entity 销毁
  → 4 级以上士兵：造成伤害的 10% 转化为自身回血
  → 骑兵无畏中：攻击力 +10，额外吸血 15%
```

### 士兵经验与升级

```
击杀 1 个敌方士兵 → 经验 +1
经验 ≥ 100 → 升级：等级 +1，最大生命值 +30，攻击力 +6
等级 ≥ 4 → 获得 10% 吸血效果（造成伤害的 10% 转化为自身生命恢复）
等级无上限
```

### 步兵手动格挡

```
框选步兵 → 底部工具栏显示 [🛡️格挡/正常] 按钮
批量规则：若选中步兵中有格挡和正常混存 → 统一切为格挡（先强制统一）
         → 再次点击切换 → 全部变正常（最多两次操作完成统一切换）
手动格挡效果：
  - 移速变为 15 px/s（固定值）
  - 正面 120° 扇形范围内伤害全额免疫
  - 背面享受被动格挡 40% 减伤（盾牌存在时）
```

### 弓兵多重射击

```
触发机率 = clamp(5% + 当前等级 × 0.5%, 5%, 30%)
  // 等级 1: 5.5%  |  等级 10: 10%  |  等级 50: 30%（封顶）
目标数 = rand(2..5)
触发后对范围内最近 2~5 个敌方单位各生成一支 Arrow
```

### 骑兵无畏效果

```
闪避成功时触发，持续 2s
效果：攻击力 +10，额外吸血 15%（与等级吸血叠加 = 10% + 15% = 25%）
2s 内再次闪避成功刷新持续时间
```

## 五、AI 行为

AI 每 2 秒评估一次全局状态，按优先级执行：

1. **防御评估**：己方城池血量 < 50% 则切换克制兵种，邻近城池派 50% 产出支援
2. **扩张评估**：攻击最近中立低等级城池（兵力 > 目标 MaxHealth × 1.5）
3. **进攻评估**：攻击最近敌方 ≤ 己方最高等级的城池（周边兵力 > 敌方 × 1.3）
4. **升级评估**：多余兵力（人口 > 上限 × 0.6）回城升级

周边兵力评估范围：目标城池半径 500px

## 七、游戏流程

### 状态机

```
MainMenu → Playing → Paused → Playing
                 ↘ GameOver → MainMenu
```

### 状态行为

| 状态 | 渲染 | 逻辑更新 | 输入 |
|------|------|---------|------|
| MainMenu | 主菜单 UI | 无 | 菜单交互 |
| Playing | 地图 + UI | 全部运行 | 游戏操作 |
| Paused | 地图(模糊) + 暂停菜单 | 冻结 | 菜单交互 |
| GameOver | 地图(灰色) + 结算面板 | 冻结 | 菜单交互 |

### 胜利条件

- 消灭敌方所有城池（攻占所有敌方城池即获胜）

### 启动流程

1. App 启动 → 注册所有 Plugin
2. 进入 MainMenu → 玩家点击"单人模式"
3. 生成随机地图 → 分配城池 → 摄像机定位到玩家城池
4. 进入 Playing

- **GameConfig**: map_width, map_height, min_cities, max_cities, city_min_distance, faction
- **GameState**: MainMenu | Playing | Paused | GameOver(Winner)

### Event

- **CitySelected** { entity, faction } — UI → City
- **CityCaptured** { entity, new_faction } — City → Game
- **SoldierSpawned** { entity, city_entity } — City → Soldier
- **SoldierDied** { entity, killer_entity } — Combat → City（击杀计数）

## 九、Bevy 系统调度顺序

