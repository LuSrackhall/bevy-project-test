## Why

当前索敌 UI 在底部工具栏平铺了 5 个按钮（全体开/步兵开/弓兵开/骑兵开/全关），交互粗糙，不支持自定义范围值，也缺乏命令反馈。用户无法精确控制索敌范围，也无法区分全局指令与选区指令的语义差异。需要重新设计为结构化的配置面板，支持兵种选择、范围输入和命令确认。

## What Changes

- **移除** 底部工具栏中现有的 5 个 `SeekBtn` 按钮和 `SeekStatusText` 状态文字
- **新增** 底部工具栏右侧索敌配置面板，根据是否选中单位自动切换两种模式：
  - **全局模式**（无选中单位）：兵种下拉选择框（全体/步兵/弓兵/骑兵）+ 范围输入框（默认 10）+ 下发按钮
  - **选择模式**（选中士兵）：兵种下拉选择框（默认全体）+ 范围输入框（默认 30）+ 下发按钮
- **新增** 范围输入交互：点击输入框进入编辑模式，键盘数字键输入，Backspace 删除，Enter 确认，Escape 取消编辑
- **新增** 兵种下拉选择框组件：点击显示当前选项，展开/收起选项列表，点击选项选中
- **新增** 顶部消息提示系统：选中单位时显示摘要，下发命令后显示确认详情
- **保留** 现有 `seek_stance_shortcut_system`（S 键快捷下发）不变
- **保留** 现有 `seek_button_system` 的命令生成逻辑，改为由新面板的按钮触发

## Capabilities

### New Capabilities

- `seek-panel`: 底部工具栏的索敌配置面板，包含兵种下拉选择框、范围输入框、下发按钮，支持全局/选择两种模式切换
- `toast-message`: 顶部消息提示系统，展示选中摘要和命令下发确认信息

### Modified Capabilities

（无已有 spec 级别的行为变更）

## Impact

- **render_view/src/ui/hud.rs**: 移除旧 seek 按钮，新增索敌面板 UI 构建与更新系统
- **render_view/src/ui/mod.rs**: 注册新系统
